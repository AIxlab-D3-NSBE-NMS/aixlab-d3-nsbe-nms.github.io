Chrony Clients Setup with Ansible

This guide explains how to configure Chrony on Linux clients using Ansible. This should be used to make sure that the camera and system times closely match between recording setups. It covers inventory setup, group variables, playbook usage, debugging, and troubleshooting tips.

Overview

Chrony is a versatile NTP client/server suite. In our lab, we standardize time synchronization using a central NTP server (ntp_server_ip). This guide ensures all clients sync correctly and consistently.

The NTP server is currently installed in the synology storage NAS.

Prerequisites

Before running the playbook, ensure:

  • Ansible 2.16+ is installed on the control node.
  • Python 3.x is installed on target hosts.
  • SSH access to all clients in the inventory.
  • The inventory and group variables are defined correctly.

Inventory & Group Variables

tree roles/chrony_client/ roles/chrony_client/ ├── handlers │   └── main.yml ├── install_chrony.yml ├── tasks │   ├── configure.yml │   ├── install.yml │   ├── main.yml │   ├── metrics.yml │   └── sync.yml └── templates └── chrony.conf.j2

Inventory Example

```ini

inventory/hosts.ini

[aixlab] aixlab-01 aixlab-02

inventory/group_vars/ntp_clients.yml

ntp_server_ip: "192.168.10.201" chrony_conf_path: /etc/chrony/chrony.conf

Tip: Always reference variables using {{ ntp_server_ip }}, not the IP directly inside curly braces.

chrony_clients.yml:

  • name: Configure Chrony client hosts: aixlab become: yes roles:
    • chrony_client

example task: - name: Ensure Chrony configuration points to the correct NTP server ansible.builtin.lineinfile: path: "{{ chrony_conf_path }}" regexp: "^server" line: "server {{ ntp_server_ip }} iburst" state: present

  • name: Restart chrony service ansible.builtin.service: name: chronyd state: restarted

debugging: - name: Debug ntp_server_ip before using it ansible.builtin.debug: var: ntp_server_ip
